## 📌 Bottom Line for FTBI

- **Agent discovery just became a real architecture topic.** Microsoft ARD + GitHub Agent Finder + Google A2A point toward governed capability catalogs: agents should discover approved tools/resources at runtime instead of hard-wiring every connector. For a Fabric/Power BI portal AI assistant, that suggests designing a small internal registry of approved report, semantic-model, alert, and documentation tools.
- **Copilot’s AGENTS.md and per-turn credit signals are worth copying.** A BI assistant should read repo/workspace-level instructions for tone, data rules, and escalation paths, and it should log cost/adoption by user and feature from day one. That keeps the POC governable when usage grows.
- **Foundry Hosted Agents are the Microsoft-native deployment path to watch.** Per-session isolation, managed identity, Application Insights traces, versioning, and scale-to-zero map cleanly to enterprise BI assistant needs: query safely, keep session state, observe behavior, and roll back bad versions without custom platform work.
